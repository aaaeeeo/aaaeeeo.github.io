Minimum Spanning Tree Problem

  • input: connected, undirected weighted graph G=(V,E)
  • output: A spanning tree T

Kruskal’s Algorithm

KRUSKAL(G,m)

O(VE) : slow

Prm’s Algorithm

  1. select a “root”
  2. grow a tree from the root by selecting the least-weight edge with one vertex already in tree that building

correctness : use “cut”

  • DEF: A cut is any partition of vertices V into two nonempty sets, S and V-S.
  1. The two cells of set $C_1$ and $C_2$ are disjoint, $C_1\cap C_2=\emptyset$
  2. $C_1\cup C_2=V$
  • CUT Property:
  1. Let G=(V,E) be a connected, undirected, weighted graph.
  2. Suppose edges $X\subseteq E$ are part of an MST T of G
  3. pick any nonempty subset S of vertinces V, $S\subseteq V$, such that no edge in X crosses between S and V-S
  4. Let $e\in E$ be an edge of minimum weight across this CUT(S, V-S), then $X\cup{e}$ is part of some MST of G, i.e. $X\cup{e}\subseteq T’$, where T’ is an MST of G
  • proof:

    edges X are part of some MST T of G. e=(u,v), edge of min weight across cut

  1. CASE 1: if the new edge $e\in T$ also, then there is nothing to prove
  2. CASE 2: assume $e\notin T$, will construct a different MST T’ containing $X\cup{e}$
    add edge e to T: $T\cup{e}$
    since T is connected and spans G, T already contains a path from u to v, so adding e=（u,v）creates a cycle
    This cycle must have some other edge e’ across CUT(S, V-S)
    if we remove this edge e’, we get $T’ = T \cup {e}-{e’}$ show T’ is a tree
    T’ is connected, since T is connected and e’ is a cycle edge
    T’ has the same number of edges as T, so T’ is a tree.
    Moreover, T’ is an MST
    Because w(T’)=w(T)+w(e)-w(e’), both e and e’ cross the CUT(S,V-S), and e is epcifically the min weight edge of this type
    So, w(e)<=w(e’), w(T’)<=w(T) -> w(T’)=w(T)
    Thus, T’ is also an MST
