Priority Queue

DEF

  1. maintains a set S of elements, each associated with a numberic key value (d-values)
  2. Supports the following operation
  • ISERT(S,x): insert element x into S
  • MIN(S): Return element of S with smallest key
  • EXTERT-MIN(S): return element of S with smallest key and removes ot from S
  • DECREASE-KEY(S, x, k): decreases the value of x’s key to new value k

Implementations

  1. ARRAY:
  • unordered array of key values for all elements
    Initially key values set to $\infty$
  • INSERT or DECREASE-KEY: O(1)
    EXTERT-MIN: linear-time scan of array
  1. BINARY HEAP:
  • A binary tree (Almost complete)
    represented as an array

Heap as binary tree

-with n vertices, tree height O(lgn)

  • root: first element in array (i=1) A[i]
    PARENT(i)= $\lfloor \left.\frac i2\right. \rfloor$
    LEFT(i)= 2i
    RIGHT(i)= 2i+1
  • min-healp property
    key[PARENT(i)]<=key[i], for all i<1 other than root
    A[PARENT(i)]<=A[i]
  • INSERT:
    MIN-HEAP-INSERT(A, key)

O(lgn)

  • DECREASE-KEY:
    HEAP-DECREASE-KEY(A, i, key)

O(lgn)

MST: Minimum Spanning Tree

  • input: Connected, undirected, weighted G=(V,E)
    with weight function w, no constraint
  • output: a spanning tree T(connects all vertices) of minimum weight
    $$ w(T)=\sum_{(u,v)\in E} w(u,v)$$

Kruskal’s ALG (Greddy)

  1. sort edges by weight
  2. choose any edge with smallest weight
    put it into spanning tree building X
  3. successively consider remaining edges in order of increasing weight
    add edge to X if does not create cycle with other edges in X
  4. stop after |v|-1 edges added to X
