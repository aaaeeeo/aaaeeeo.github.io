Linux常用命令整理

  • linux
  • 整理
linux

Linux

man [command]
查看一个linux命令的说明文档

文件/目录修改

  1. cd

    . 当前目录
    .. 上级目录

  2. ls

    -l ：列出长数据串，包含文件的属性与权限数据等
    -a ：列出全部的文件，连同隐藏文件（开头为.的文件）一起列出来
    -d ：仅列出目录本身，而不是列出目录的文件数据
    -h ：将文件容量以较易读的方式（GB，kB等）列出来
    -R ：递归列出子目录

  3. cp [options] source1 source2 source3 …. directory

    -a ：将文件的特性一起复制
    -p ：连同文件的属性一起复制，而非使用默认方式，与-a相似，常用于备份
    -i ：若目标文件已经存在时，在覆盖时会先询问操作的进行
    -r ：递归复制目录
    -u ：目标文件与源文件有差异时才会复制

  4. mv [options] source1 source2 source3 …. directory

    -f 强制 -i 询问
    -u ：若目标文件已经存在，且比目标文件新，才会更新

  5. rm

    -f 强制 -i 询问 -r 递归

  6. touch
    建立空文件，或者将某个文件访问时间修改为目前(mtime 与 atime)

    -t [YYMMDDhhmm] ：修改为指定时间

  7. mkdir

    -m ：设定目录权限
    -p ：递归建立目录 （mkdir -p dir1/dir2）

  8. tar
    压缩/解压

    -j ：通过bzip2的支持进行压缩/解压缩
    -z ：通过gzip的支持进行压缩/解压缩

    压缩：tar -zcv -f filename.tar.gz 要被处理的文件或目录名称
    查询：tar -ztv -f filename.tar.gz
    解压：tar -zxv -f filename.tar.gz -C 欲解压缩的目录

文件/信息查看

  1. cat tac（从最后一行开始显示）nl（有行号显示）

    -n ：打印出行号；
    -T ：将 [tab] 按键以 ^I 显示出来；
    -v ：列出一些看不出来的特殊字符

  2. head / tail

    -n ：后面接数字，代表显示几行

  3. less
    分页显示，例： cat 1.cpp | less

    空格键 ：向下翻动一页；
    b ： 向上翻动一页；
    Enter： 向下翻一行
    上/下键盘：上下翻动一行
    /字符串 ：向下搜寻『字符串』的功能；
    ?字符串 ：向上搜寻『字符串』的功能；
    n ：重复前一个搜寻 (与 / 或 ? 有关！)
    N ：反向的重复前一个搜寻 (与 / 或 ? 有关！)
    q ：离开 less 这个程序

  4. stat
    查看文件基本信息（大小、权限、日期等）

  5. file
    查看文件信息（类型、链接库等）

  6. diff [文件或目录1] [文件或目录2]
    以逐行的方式，比较文本文件的异同处。所是指定要比较目录，则diff会比较目录中相同文件名的文件

    -y或–side-by-side 　以并列的方式显示文件的异同之处
    -u,-U<列数>或–unified=<列数> 　以合并的方式来显示文件内容的不同。
    -b或–ignore-space-change 　不检查空格字符的不同。
    -B或–ignore-blank-lines 　不检查空白行。
    -c 　显示全部内文，并标出不同之处。

  1. grep ‘查找字符串’ filename
    分析一行的信息，查找指定内容

    -a ：将binary文件以text文件的方式查找数据
    -c ：计算找到‘查找字符串’的次数
    -i ：忽略大小写的区别，即把大小写视为相同
    -v ：反向选择，即显示出没有‘查找字符串’内容的那一行
    例：
    grep ‘MANPATH’ /etc/man.config
    ps | grep -i mysql

  2. find [PATH] [option] [action]
    find是一个基于查找的功能非常强大的命令

    [PATH]: 查找路径
    [option]: 查找内容
    [action]: 对找到内容执行的动作，-print或 -exec ‘command’ {} \;
    例：
    find ./ -name file1 -print ,从当前目录向下查找名为 file1 的文件并打印出来
    find ./ -size 0 -exec rm {} \; 删除文件大小为零的文件（还可以以这样做：rm -i `find ./ -size 0` 或 find ./ -size 0 | xargs rm -f &）
    find /root/py -mtime +10 -name “*.dmp” -exec cp {} /root/davetmp/ /;

  • 与时间有关的参数：
    -mtime n : n为数字，意思为在n天之前的“一天内”被更改过的文件；
    -mtime +n : 列出在n天之前（不含n天本身）被更改过的文件名；
    -mtime -n : 列出在n天之内（含n天本身）被更改过的文件名；
    -newer file : 列出比file还要新的文件名
    例如：
    find /root -mtime 0 #在当前目录下查找今天之内有改动的文件

  • 与用户或用户组名有关的参数：
    -user name : 列出文件所有者为name的文件
    -group name : 列出文件所属用户组为name的文件
    -uid n : 列出文件所有者为用户ID为n的文件
    -gid n : 列出文件所属用户组为用户组ID为n的文件
    例如：
    find /home/ljianhui -user ljianhui #在目录/home/ljianhui中找出所有者为ljianhui的文件

  • 与文件权限及名称有关的参数：
    -name filename ：找出文件名为filename的文件
    -size [+-]SIZE ：找出比SIZE还要大（+）或小（-）的文件
    -tpye TYPE ：查找文件的类型为TYPE的文件，TYPE的值主要有：一般文件（f)、设备文件（b、c）、 

    目录（d）、连接文件（l）、socket（s）、FIFO管道文件（p）；

    -perm mode ：查找文件权限刚好等于mode的文件，mode用数字表示，如0755；
    -perm -mode ：查找文件权限必须要全部包括mode权限的文件，mode用数字表示
    -perm +mode ：查找文件权限包含任一mode的权限的文件，mode用数字表示
    例如：
    find / -name passwd # 查找文件名为passwd的文件
    find . -perm 0755 # 查找当前目录中文件权限的0755的文件
    find . -size +12k # 查找当前目录中大于12KB的文件，注意c表示byte

权限相关

  1. chmod

    -R 递归修改

    chmod 755 file #把file的文件权限改变为-rxwr-xr-x
    755 => 对应8进制的 111 可读可写可执行

    chmod g+w file #向file的文件权限中加入用户组可写权限
    u（user）、g（group）、o（other）、a（all）和+（加入）、-（删除）、=（设置）跟rwx搭配来对文件的权限进行更改。

  2. chgrp
    改变文件所属用户组

    chgrp users -R ./dir

  3. chown
    改变文件所有者

  4. umask
    查看或更改当前默认权限

    umask 022

  5. su
    变更为其他使用者的身份

    -l 或 –login 这个参数加了之后，就好像是重新 login 为该使用者一样
    -m -p 或 –preserve-environment 执行 su 时不改变环境变数
    例：
    su root

  6. sudo

    -l 显示出自己（执行 sudo 的使用者）的权限
    -u username/#uid 不加此参数，代表要以 root 的身份执行指令，而加了此参数，可以以 username 的身份执行指令（#uid 为该 username 的使用者号码）

系统

  1. which
    查看command所在位置 例：which python

    -a ：将所有可以找到的指令均列出，而不止第一个被找到的指令名称

  2. top
    任务管理器

  3. ps
    查看进程

    -A ：所有的进程均显示出来
    -a ：不与terminal有关的所有进程
    -u ：有效用户的相关进程
    -x ：一般与a参数一起使用，可列出较完整的信息
    -l ：较长，较详细地将PID的信息列出

  4. jobs / bg / fg

    ctrl-c 发送 SIGINT 信号，终止一个进程
    ctrl-z 发送 SIGSTOP信号，挂起一个进程
    ctrl-d EOF，终止输入流

    jobs: 查看后台任务
    bg: 后台运行任务
    fg: 前台运行任务
    bg %任务编号 默认第一个

  5. kill
    向进程发送信号，默认SIGKILL强制中断信号

    kill [-signal] PID或%jobnumber

  6. who / finger
    查看在线用户/用户信息

  7. fdisk
    列出存储设备
